The World Wide Web

Welcome to the intro of the World Wide Web. On this web page we will give you a brief history on the www aswell as cover when and by whom the www was created and basic principles regarding how it works in todays world.

What is the world wide web?

The world wide web is a network which contains a range of information working in tandem with hypertext to give people easy and powerful access to a global information system. Hypertext is a document that contains addresses/links to other parts of the document or a totally new document, this gives users easy access to information. The www ( world wide web ) is not the same as the Internet but rather a portion of it, thus we can say the www runs/operates within the Internet. When searching for information we use a URL or Uniform Resource Locator. The URL contains three elements that help it find information. One of the protocols used called HTTP is for web pages (click here to learn more ) then we need the IP address or Domain Name of the server that holds the information and then to make searching for information even easier the Pathname for the file is optional. These three elements work in tandem to locate and request data for the user.

Difference between the WWW and the Internet

The Internet is simply a globally connected network which gives us access to resources for information. Whereas the World Wide Web gives us access to a huge array of documents which are connected to each other via hypertext or hypermedia links. If you think of the internet and the WWW in a very advanced level, you can say that the internet is the hardware and the WWW is the software. The WWW was only founded in 1989 whereas the internet was found in the late 1960’s. The internet uses IP addresses as identification, whereas the WWW uses Hypertext Transfer Protocol (HTTP).

Evolution Of The Internet

ARPAnet

J.C.R. Licklider, an MIT computer scientist, proposes the concept of a worldwide computer network in 1962. Later, he discusses his concept with colleagues at the Advanced Research Projects Agency (ARPA) of the United States Department of Defense. The ARPAnet was first published in October 1969.

TCP/IP

Robert Kahn and Vinton Cerf collaborated in 1973 to establish a protocol for connecting numerous networks. This subsequently evolves into the Transmission Control Protocol/Internet Protocol (TCP/IP), a system that connects many networks such that if one goes down, the others don't.

3Com

3Com releases Ethernet devices for both computer workstations and personal computers in 1981, allowing local area networks to be established (LANs).

PhoneNet System

The PhoneNet system is developed in 1982 and is connected to ARPANET and Telenet, the first commercial network. This increases internet availability and enables for email communication across countries around the globe

Domain Name System

The first domain was registered in 1985, following the creation of the Domain Name System by Paul Mockapetris, Jon Postel, and Craig Partridge, which utilizes domain names to handle the growing number of internet users

Gateway-to-Gateway Protocol

In 1987, the Gateway-to-Gateway Protocol (GGP) was created to route traffic across networks (rather than within a single network), paving the way for the Border Gateway Protocol (BGP), which was established to share routing and reachability information among autonomous systems (AS) on the Internet.

HTML and URLs

ARPANET is discontinued in 1990. The World Wide Web is born when Tim Berners-Lee and his colleagues at CERN create the hypertext markup language (HTML) and the universal resource location (URL).

"Internet"

The Federal Networking Council unanimously passed a resolution on October 24, 1995, that defined the word "internet." Members of the internet and intellectual property rights communities contributed to the development of this concept.

...

Between 1994 and 1995, Microsoft releases Windows 95, Amazon, Yahoo, and eBay all launch, Internet Explorer is released, and Java is developed, allowing for website animation and sparking a new wave of internet activity.

Protocols used on the internet

TCP

Transmission Control Protocol

Transmission Transfer Protocol is a popular communication protocol. It is used to communicate over a network.

IP

Internet Protocol

Internet Protocol is the most popular protocol. It acts as a address protocol and is mostly used inline with TCP allowing the packets sent with TCP to know their destination.

UDP

User Datagram Protocol

User Datagram Protocol is a substitute to TCP that offers low latency and loss tolerating communication between applications

POP

Post Office Protocol

Post Office Protocol is used for receiving emails.

SMTP

Simple Mail Transfer Protocol

Simple Mail Transfer Protocol is used for sending and distributing emails.

FTP

File Transfer Protocol

File Transfer Protocol allows users to transfer files between devices.

HTTP

Hypertext Transfer Protocol

Hypertext Transfer Protocol is used to transfer hypertext documents between machines.

HTTPS

Hypertext Transfer Protocol Secure

Hypertext Transfer Protocol Secure is used to transfer hypertext documents between machines with a extra security layer.

Client-Side and Server-Side Engineering

What is Client- and Server-Side?

These are basic web development terms to describe where the application code runs. Developers will sometimes also refer to this as frontend and backend.

What is the Client-Side?

This is when a device uses a network to access centrally located servers to get the data they need, instead of communicating with each other. End user devices, such as your laptop or smartphone is 3 considered to be clients of the servers. They simply send requests to the servers to receive the data, and then the server sends them the data. The client-side refers to everything that takes place or is being processed on the user’s device(client- side).
